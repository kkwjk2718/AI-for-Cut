# 보안 정책

## 지원 범위

현재 `main` 브랜치의 최신 버전을 기준으로 보안 문제를 확인합니다.

## 신고 방법

보안 취약점은 공개 Issue에 올리지 마세요. GitHub의 비공개 보안 신고 기능이 열려 있으면 해당 기능을 사용하고, 그렇지 않으면 저장소 관리자에게 직접 알려 주세요.

신고 시 가능한 범위에서 아래 정보를 포함해 주세요.

- 영향받는 기능 또는 API 경로
- 재현 절차
- 예상 영향
- 로그나 스크린샷이 있다면 개인정보를 제거한 자료

## 민감 정보

다음 정보는 저장소, Issue, PR, 로그에 올리지 않습니다.

- OpenAI API 키
- Brevo API 키
- `ADMIN_PIN`, `CRON_SECRET`
- 참가자 사진과 이메일 주소
- 운영 환경의 `.env`

민감 정보가 노출되었다면 즉시 해당 키를 폐기하고 새 키를 발급하세요.
